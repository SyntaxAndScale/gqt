# Gqueues TUI (gqt) Directives

- **Code Integrity:** Always run `cargo check` after writing code to ensure it compiles before requesting review.
- **Git Policy:** Always create a new git branch when beginning work on a feature or significant code change. Never commit to git without explicit user permission.
- **Versioning:** Only increment semver when explicitly instructed. Favor patch increments (e.g., 0.1.0 -> 0.1.1) over minor or major.
- **Documentation:** Always append to `journal.md` when work is done, decisions are made, or commits are authorized.
- **Context Efficiency:** Strive to keep token usage concise and efficient.
- **Task Management:** Log every new directive in `todo.md`.
- **Safety:** Never hardcode API keys or secrets. Use environment variables or configuration files.
